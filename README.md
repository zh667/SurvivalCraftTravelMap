# Survivalcraft Travel Map

Survivalcraft Travel Map（SCTM）是面向 Survivalcraft 联机版 2.4.40.6 / NetMod API 1.44 的旅行地图 Mod。它在单人、主机和联机客户端显示持久化探索地图，并提供安全坐标传送、完整 XYZ 坐标点和玩家邀请传送。

## 安装

1. 退出游戏。
2. 从 `NetMods` 中移除 `34GPSFix.netmod`。SCTM 与它不能同时安装；若检测到冲突，SCTM 会提示并停止初始化。
3. 将 `SurvivalcraftTravelMap.netmod` 放入游戏目录的 `NetMods` 文件夹。
4. 启动游戏并进入世界。

远程联机坐标传送要求服务器也安装相同版本的 SCTM。只在客户端安装时仍可查看地图，但地图右键传送和坐标点传送不会由客户端直接修改玩家位置。联机版的单人世界实际以 `WorkType.Server` 集成主机运行；主玩家仍会创建地图界面，坐标传送则在同一进程内由服务器权威执行。

## 操作

- 右上角小地图：显示已探索地形、玩家方向、坐标点和可选 XYZ 坐标。位置按玩家 `GuiWidget.ActualSize` 的逻辑坐标计算，不使用物理视口像素，因此会跟随游戏 UI 缩放。
- `M`：打开或关闭大地图；聊天输入、文本输入或模态对话框占用焦点时不会触发。
- 鼠标滚轮：光标位于地图上时缩放。
- 按住鼠标左键拖动：平移大地图。
- 右键已探索区域：传送到附近安全地面，或选择“保存当前位置”把玩家此刻的真实 XYZ 保存为坐标点。右键位置不会被伪造为一个远处 Y。
- 右键坐标点：传送、重命名或删除。
- 大地图“设置”：开关小地图、坐标、日夜明暗和邀请；调整小地图尺寸与两种地图缩放。
- 玩家邀请按钮：打开玩家列表，每页 4 人；旧 ID 41 邀请协议保持兼容。

未探索区域不会被地图渲染或允许右键传送。地图地形随日夜变暗，玩家箭头、坐标点和文字保持可读亮度。

## 安全传送

联机版单人世界由集成主机直接执行服务器权威安全传送：

- 目标区块最多等待 10 秒。
- 在目标水平半径 8 格内寻找实体地面与连续两格空气。
- 拒绝水、岩浆、火、仙人掌、尖刺、树叶、下落方块及其他危险/碰撞位置。
- 最终位置落在方块中心，并清除速度和坠落状态。
- 移动后一帧再次验证；失败时回退到原位置的安全静止状态。

坐标点保存完整 XYZ。传送到坐标点时只围绕保存的 Y 上下各搜索 8 格，不会改用地表高度；因此洞穴坐标点不会把玩家送到山顶。

远程联机中，客户端只发送目标 XZ 或 XYZ；服务器重新加载区块、计算安全 Y 并执行传送。远程 ID 61 协议的客户端响应窗口为 5 秒，服务器执行截止时间为 4 秒；截止或掉线会取消并回退未完成事务，避免客户端已超时后又发生迟到传送。集成主机不经过这条远程协议期限，安全搜索可使用完整的 10 秒区块等待上限。服务器确认传送成功后会广播 `PositionSet`，使已连接客户端同步服务器上的权威位置。服务器未安装 SCTM、禁用该功能或无能力响应时，远程客户端每次会话只提示一次“服务器不支持地图传送或请求超时”。

## 数据与备份

所有路径均位于游戏的 `app:/SurvivalcraftTravelMap`：

- 全局界面设置：`settings.json`（`schemaVersion: 1`；旧 `travel-map-settings.json` 会一次性迁移且保留原文件）
- 每世界、每玩家地图：`maps/<world-key>/<player-guid>/tiles/*.sctm`
- 每世界、每玩家坐标点：`maps/<world-key>/<player-guid>/waypoints.json`
- 服务器开关：`server-settings.json`

本地世界键由规范化世界目录生成；联机世界键由服务器地址、端口和服务器世界标识生成。玩家目录使用玩家 GUID，避免不同世界、服务器或玩家的数据混用。若可靠身份、存储目录或初始地图/坐标点持久化无法建立，该玩家组件保持未激活，不创建地图 UI，并显示提示；游戏本身继续运行。

探索记录只接受状态为 `TerrainChunkState.Valid` 的区块，并且只有非透明采样颜色才会写为已探索。格式版本 1 将 `A=0` 保留给空气/不可用采样哨兵；旧缓存中“探索位已设置但颜色透明”的像素会自动按未探索处理，区块有效后可重新采样，不需要手动删除地图目录。

一般设置文件读取失败时会使用只读的安全默认设置继续初始化；检测到更高版本的 `settings.json` 时，本版本同样只在内存中使用安全默认值并提示一次，不会归一化、保存或覆盖原文件。地图已成功初始化后，若后续瓦片写入持续失败，缓存达到上限会暂停接收新区块探索并提示一次；已有脏瓦片继续保留和重试，成功刷新后自动恢复。

备份时退出游戏，然后复制整个 `SurvivalcraftTravelMap` 数据目录。卸载 Mod 前也建议这样做。

## 服务器设置

首次以服务器模式启动后会创建：

```json
{
  "schemaVersion": 1,
  "surfaceTeleportEnabled": true,
  "waypointTeleportEnabled": true
}
```

修改后重启服务器。未来版本的未知 schema 会以只读方式处理，并使用安全默认值，避免覆盖新格式。

## 卸载

1. 退出游戏并备份 `app:/SurvivalcraftTravelMap`。
2. 从 `NetMods` 删除 `SurvivalcraftTravelMap.netmod`。
3. 若不再需要探索记录和坐标点，可手动删除 `app:/SurvivalcraftTravelMap`；卸载 Mod 本身不会删除这些数据。

## 隐私与兼容性

SCTM 不扫描、统计或上报已安装 Mod 的数量，也不包含旧的 Mod 数量校验/上报协议。它只注册玩家邀请 ID 41 和服务器权威坐标传送 ID 61。

已知限制：

- 当前仅针对 Windows PC、Survivalcraft 2.4.40.6 与 NetMod API 1.44 构建。
- 联机坐标传送要求服务器安装 SCTM；旧服务器只能使用其已有的邀请传送能力。
- 全细节地图在 64 格瓦片接缝处，探索边界青线可能出现一小段不连续；探索数据本身不受影响。
- 本轮针对小地图逻辑坐标、有效区块重采样和集成主机传送链路的修复已通过 439/439 全量测试、零警告严格构建和封包校验，但仍待第二轮实机复测，不能视为最终验收完成。
- 实际游戏内布局、各地形和主机/客户端场景仍应按 [烟雾测试清单](docs/smoke-test-2026-07-13.md) 完成人工确认。

更完整的操作、故障恢复与测试步骤见 [用户指南](docs/user-guide.md)。
