# Survivalcraft Travel Map 设计规格

状态：已获用户批准  
日期：2026-07-13  
目标平台：Survivalcraft 联机版 2.4.40.6 / NetMod API 1.44 / Windows PC  

## 1. 项目目标

以 `NetMods/34GPSFix.netmod` 的副本为技术底座，重建一个独立、可维护的 NetMod。保留原有小地图、玩家列表和邀请传送能力，删除 Mod 数量扫描与上报，新增以下能力：

- 小地图缩放和尺寸调整。
- 按 `M` 打开大型旅行地图。
- 已探索区域持久化。
- 实时 XYZ 坐标显示。
- 地图日夜明暗变化。
- 在地图任意已探索位置右键安全传送。
- 创建、重命名、删除带完整 XYZ 的坐标点。
- 右键坐标点安全传送，支持地下住宅、矿洞等保存 Y 的位置。
- 单人和联机模式均显示地图传送；联机执行必须由安装本 Mod 的服务器验证。

原始 `34GPSFix.netmod` 必须保持字节不变。开发、打包和测试均使用副本，最终产物使用新文件名和新包身份。

## 2. 产品身份

- 显示名：`Survivalcraft Travel Map`
- 简称：`SCTM`
- 包名：`SurvivalcraftTravelMap`
- 程序集名：`SurvivalcraftTravelMap.dll`
- 输出文件：`SurvivalcraftTravelMap.netmod`
- 默认作者字段：`SCTM`
- XDB 实体和组件全部使用新 GUID。
- 不允许与 `34GPSFix.netmod` 同时安装，启动时检测到旧包应显示明确冲突提示并停止初始化 SCTM。

公开清单和游戏界面不强制显示 `tk34` 或 `34GPSFix` 署名。由于代码来源是现有二进制的派生改造且原包未声明许可证，公开分发前必须由发布者确认其拥有相应授权；在此之前不得宣称拥有全部原始代码的版权，也不得擅自为全部派生代码套用 MIT 等许可证。

- 普通玩家邀请传送、同意、拒绝和 30 秒超时。
- 多服务器玩家列表界面的兼容入口；空实现不得伪装成已支持能力。
- 旧邀请协议的数据包 ID `41` 和既有六种消息格式，以尽量保持服务器兼容。

### 3.2 本次删除

- `AntiCheatReportPackage`。
- `.netmod` 文件数量扫描。
- `ReadOnlyModList` 和 `ReadOnlyModListAll` 数量统计。

## 4. 架构与模块边界

### 4.1 `TravelMapModLoader`

- 注册 XDB、旧邀请协议和新的坐标传送协议。
- 创建全局服务并负责释放后台保存任务。
- 检测 `34GPSFix` 冲突。
- 不扫描或上报用户安装的其他 Mod。

### 4.2 `TravelMapComponent`

- 挂载到 Player，只为主玩家绘制和更新地图。
- 组合地图视图、探索记录器、设置和传送入口。
- 服务器进程不绘制 UI；本地世界和联机客户端绘制 UI。

### 4.3 `TerrainMapSampler`

- 根据 X/Z 读取最高有效地形方块。
- 跳过不应覆盖地图的表层装饰。
- 将水类特殊方块归一化。
- 根据季节温度、海拔温度和湿度计算草、树叶、水等颜色。
- 输出不包含日夜明暗的基础颜色；日夜效果只在绘制阶段叠加，避免污染持久化瓦片。

### 4.4 `MiniMapRenderer`

- 绘制右上角小地图、玩家箭头、坐标文字和坐标点。
- 只消费已探索瓦片，不主动显示未知地形。
- 处理鼠标悬停时的滚轮缩放。

### 4.5 `TravelMapDialog`

- 按 `M` 打开。
- 仅在没有文本输入焦点、聊天输入或其他阻塞模态窗口时响应快捷键。
- 支持滚轮缩放、按住左键拖动、右键上下文菜单。
- 使用 `MapTransform` 完成世界坐标与屏幕坐标转换。

### 4.6 `ExplorationRecorder` 与 `ExplorationTileStore`

- `ExplorationRecorder` 在玩家移动或地形刷新时记录可见区域。
- `ExplorationTileStore` 负责瓦片缓存、压缩、原子写入、载入和损坏隔离。
- 两者不直接依赖 UI，便于自动测试。

### 4.7 `WaypointRepository`

- 保存完整 XYZ、名称、创建时间和稳定 ID。
- 支持新增、重命名、删除、查询和原子保存。
- 不允许空名称；同名坐标点允许存在，但 UI 显示坐标以便区分。

### 4.8 `SafeTeleportService`

- 负责区块加载、安全候选生成、危险判断、执行、二次验证和失败回退。
- 通过 `ITerrainAccess`、`IChunkLoader` 和 `IPlayerMover` 接口隔离游戏引擎，核心算法可脱离游戏程序集测试。

### 4.9 旧玩家邀请模块

- 保留并整理 `GPSPackage`、`ITPManager`、`Invitation`、`TeleportPanelWidget` 和玩家条目。
- 修复设置开关关闭后连普通提示也被屏蔽的问题：只拒绝新的邀请弹窗，普通结果提示继续显示。
- 玩家传送列表仍每页显示 4 人。

## 5. 小地图规格

- 默认显示尺寸：`256 x 256`。
- 可选尺寸：`160 / 192 / 256 / 320 / 384`。
- 默认位置：右上角，顶部边距 32，右侧边距 100。
- 地图保持世界方向固定，只旋转玩家箭头。
- 玩家箭头默认 `32 x 32`，随小地图尺寸按比例限制在 `24..40`。
- 小地图缩放以“每像素对应方块数”表示，范围 `0.5..8`，滚轮每档乘或除 `sqrt(2)`，结果钳制到范围内。
- 设置窗口支持开关：显示小地图、显示坐标、启用日夜明暗、接受玩家邀请。
- 坐标格式：`X: 123  Y: 64  Z: -456`。
- 玩家移动时更新附近探索数据；静止时最多每 0.5 秒重新采样一次可见区域。
- 未探索像素使用半透明深灰，不读取其地形来“透视”。

## 6. 大型旅行地图交互

- 快捷键：`M`。
- 打开后默认以玩家为中心。
- 大地图缩放范围：`0.25..32` 方块/像素。
- 鼠标滚轮以指针位置为缩放中心。
- 按住左键拖动浏览。
- 玩家位置使用方向箭头。
- 坐标点使用独立图标和名称标签；标签重叠时优先显示离鼠标最近者。

右键普通已探索位置显示：

1. `传送到这里`
2. `添加坐标点`
3. `取消`

右键坐标点显示：

1. `传送到此坐标点`
2. `重命名`
3. `删除`
4. `取消`

右键未探索区域不允许传送或新增坐标点，显示“该区域尚未探索”。

## 7. 探索区域持久化

### 7.1 瓦片模型

- 每个地图瓦片覆盖 `64 x 64` 个世界方块。
- 每个像素保存 RGBA 基础地形颜色，并保存独立探索位图。
- 负坐标使用数学向下取整划分瓦片，确保 `-1` 属于瓦片 `-1` 而不是 `0`。
- 文件头为 ASCII `SCTM`，格式版本为 `1`。
- 文件内容使用 Deflate 压缩，包含坐标、探索位图、颜色数据和校验值。
- 校验失败的单个瓦片改名为 `.corrupt` 并视为未探索，不影响其他瓦片。

### 7.2 保存策略

- 发生变化的瓦片标记为 dirty。
- dirty 瓦片最多每 5 秒批量保存一次。
- 退出世界和 Mod 释放时执行最后一次有界保存。
- 写入 `*.tmp`，完成刷新后原子替换正式文件。
- 内存 LRU 默认保留最近 128 个瓦片；dirty 瓦片写入成功前不得淘汰。

### 7.3 数据隔离与路径

根路径：`app:/SurvivalcraftTravelMap/`

- 设置：`settings.json`
- 地图数据：`maps/<world-key>/<player-guid>/tiles/<tile-x>_<tile-z>.sctm`
- 坐标点：`maps/<world-key>/<player-guid>/waypoints.json`

`world-key` 规则：

- 单人世界：规范化世界目录标识的 SHA-256 前 24 个十六进制字符。
- 联机世界：规范化服务器主机名、端口和服务器世界标识组合后的 SHA-256 前 24 个十六进制字符。

所有 JSON 均包含 `schemaVersion: 1`。未知更高版本只读保护，不覆盖原文件。

## 8. 日夜明暗

- 持久化瓦片只保存白天基础颜色。
- 绘制时读取当前世界时间，计算平滑亮度系数。
- 正午系数 `1.0`，深夜最低 `0.4`，日出日落连续过渡。
- 玩家箭头、坐标文字、坐标点图标和 UI 不受地形明暗影响。
- 设置中可关闭日夜明暗；关闭后系数恒为 `1.0`。

## 9. 安全传送算法

### 9.1 通用流程

1. 保存原始玩家位置、速度和可恢复状态。
2. 请求加载目标及周围 8 格涉及的区块，最长等待 10 秒。
3. 超时、区块无效或世界边界非法时取消，不移动玩家。
4. 按传送类型生成候选位置。
5. 对候选执行安全验证。
6. 选择距离目标最近且评分最高的位置。
7. 把玩家放到方块中心 `X + 0.5, Z + 0.5` 和安全地面上方。
8. 清零线速度和角速度，并重置可用的坠落伤害累计状态。
9. 下一更新帧重新验证脚下、身体和头部空间。
10. 二次验证失败时立即回退到原位置并清零速度。

### 9.2 普通地图位置

- 地图点击只提供 X/Z。
- 在目标周围 Chebyshev 半径 8 内枚举水平候选，按水平距离、危险评分排序。
- 每一列从可靠地表高度开始验证。
- 不为了找到落点而向玩家暴露未探索地形；只有点击点已探索时才允许开始。

### 9.3 已保存坐标点

- 使用保存的完整 XYZ。
- 先验证原坐标。
- 再按 `0, +1, -1, +2, -2 ... +8, -8` 的垂直顺序，并结合水平半径 8 搜索。
- 不自动改用遥远地表，以免地下“家”被传到地面。
- 搜索体内没有安全位置则取消传送。

### 9.4 安全位置判定

- 脚下方块必须可碰撞、可站立且不是空气。
- 玩家身体和头部占用的两格必须无实体碰撞。
- 岩浆、火、尖刺、仙人掌及具有持续伤害属性的方块无条件拒绝。
- 水面、流体、树叶、易坠落方块和不稳定表面不作为有效落点。
- 候选不得位于世界高度范围外。
- 候选不得与其他实体的阻挡包围盒重叠。

## 10. 单人与联机传送

### 10.1 单人世界

- 客户端本地直接调用 `SafeTeleportService`。
- 不需要网络包。

### 10.2 联机世界

- 地图和右键菜单始终显示。
- 新增独立坐标传送协议，数据包 ID 使用 `61`，不扩展旧 ID `41`，避免旧服务器错误解析新消息。
- 请求只包含传送模式、目标 XYZ 或目标 XZ、客户端请求 ID；不包含客户端计算的最终安全 Y。
- 服务器重新执行区块加载和安全搜索，以服务器结果为准。
- 服务器返回成功、拒绝、不支持、超时、无安全落点等明确结果。
- 客户端等待服务器响应 5 秒；无响应时提示“服务器不支持地图传送或请求超时”。
- 客户端不得在无服务器确认时直接改写联机位置。
- 服务器可通过配置关闭普通地图传送、坐标点传送或两者。

## 11. 设置模型

`TravelMapSettings` 默认值：

- `IsMiniMapVisible = true`
- `ShowCoordinates = true`
- `UseDayNightTint = true`
- `AcceptTeleportInvitations = true`
- `MiniMapSize = 256`
- `MiniMapBlocksPerPixel = 1.0`
- `LargeMapBlocksPerPixel = 2.0`
- `LargeMapHotkey = M`
- `NightMinimumBrightness = 0.4`

不识别或越界的设置值恢复到最近合法值，并保存修正后的配置。旧的 `GPSSetting.xml` 不自动删除；首次迁移时只读取其中的小地图显示和邀请开关，成功迁移后写入新设置文件。

## 12. 错误处理

- 单个地图瓦片损坏只隔离该瓦片。
- 坐标点 JSON 损坏时保留原文件为 `.corrupt`，创建空仓库并提示一次。
- 地图保存失败不阻塞游戏线程；保留 dirty 状态并退避重试。
- 内存不足时优先降低缓存数量，不删除未保存瓦片。
- 传送失败永远不部分执行；成功前保持原位置，二次验证失败必须回退。
- 联机服务器不支持新协议时不刷屏，同一服务器每次会话只提示一次能力缺失。
- UI 资源缺失时使用内建文字按钮和纯色图形，不依赖缺失的 `Setting.png`。

## 13. 测试策略

所有新行为遵循测试先行的 Red-Green-Refactor 流程。

### 13.1 纯逻辑测试

- 世界坐标与瓦片坐标，特别是 `-1/-64/-65`。
- 世界坐标与屏幕坐标双向转换。
- 各缩放级别以鼠标位置为中心的缩放不漂移。
- 瓦片序列化、压缩、校验、损坏隔离和原子恢复。
- LRU 不淘汰 dirty 瓦片。
- 坐标点新增、同名、重命名、删除和重新载入。
- 日夜亮度边界和平滑过渡。
- 普通地图候选排序。
- 坐标点 Y 优先搜索顺序。
- 两格净空、危险方块、流体、树叶、世界高度和实体碰撞拒绝。
- 区块加载超时不移动玩家。
- 二次验证失败回退原位置并清零速度。

### 13.2 集成测试

- XDB 只注入一个旅行地图组件。
- 原 34GPSFix 冲突检测。
- 本地世界地图显示、探索保存、重启恢复。
- `M` 键在聊天输入时不触发。
- 联机旧邀请协议 ID `41` 保持原格式。
- 新协议 ID `61` 的服务器验证、拒绝和超时。
- 关闭“接受邀请”只屏蔽邀请，不屏蔽普通结果提示。
- Mod 包中不存在反作弊上报类、ID `60` 注册和扫描线程。

### 13.3 手动验收矩阵

- 平地、树林、山顶、洞穴、水边、岩浆附近。
- 正坐标、负坐标、瓦片边界。
- 白天、黄昏、午夜和日出。
- 单人世界、主机服务器、联机客户端、不支持 SCTM 的旧服务器。
- 不同 UI 缩放和窗口尺寸。

## 14. 验收标准

- 原始 `34GPSFix.netmod` 的 SHA-256 保持不变。
- 仅安装 `SurvivalcraftTravelMap.netmod` 时，单人世界可显示小地图和大型地图。
- 已探索地形在退出并重新进入后恢复，未探索区域不泄露地形。
- 小地图尺寸、缩放、坐标和日夜选项均持久化。
- 普通地图传送和坐标点传送不会落入实体方块、危险方块或无地面高空。
- 地下坐标点优先保留原 Y，不自动跳到地面。
- 传送加载超时和找不到安全点时玩家位置不变。
- 联机坐标传送必须经过服务器验证；旧服务器无响应时不强制传送。
- 原邀请传送仍可使用。
- 包内不存在 Mod 数量上报功能。
- 构建、自动测试、包结构检查和一次实际游戏冒烟测试全部通过后才形成可分享构建。
